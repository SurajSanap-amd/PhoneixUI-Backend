# Team Portal Backend
[.](https://phoneixui-backend.onrender.com/api/ideas/getAll)
## Overview

The **Team Portal Backend** is a Spring Boot-based application designed to manage and track **Improvement Items**. These items are suggestions, improvements, or toil reduction tasks aimed at optimizing processes and tools within the team.

This application provides RESTful APIs to create, read, update, delete, and like improvement items. It also supports exporting the improvement items into an Excel file for offline analysis or sharing.

## Features

* **CRUD Operations**: Create, Read, Update, and Delete improvement items.
* **Like System**: Users can like improvement items to indicate importance or popularity.
* **Excel Export**: Export all improvement items to an Excel file.
* **Paginated API**: Retrieve improvement items with pagination and sorting.
* **Validation**: Input validation with `@NotBlank` and other annotations.
* **Error Handling**: Graceful error handling with meaningful status codes.

## Technology Stack

* **Framework**: Spring Boot
* **Database**: Relational DB (JPA with Hibernate)
* **Build Tool**: Maven / Gradle
* **Language**: Java (JDK 17+)
* **Excel Export**: Apache POI (via custom `ExcelExporter` utility)
* **Version Control**: Git
* **API Documentation**: Swagger or manual documentation (future enhancement)

## API Endpoints

### 1. **Get All Improvement Items**

* **GET** `/api/improvements`
* **Description**: Retrieve a list of all improvement items.

### 2. **Get Improvement Items with Pagination**

* **GET** `/api/improvements/paged`
* **Query Parameters**:

  * `page`: Page number (default: 0)
  * `size`: Page size (default: 10)
  * `sortBy`: Field to sort by (default: `id`)
  * `direction`: Sort direction (`asc` or `desc`, default: `asc`)

### 3. **Create New Improvement Item**

* **POST** `/api/improvements`
* **Body**: `Idea` (JSON)
* **Description**: Create a new improvement item.

### 4. **Update Improvement Item**

* **PUT** `/api/improvements/{id}`
* **Path Variable**: `id` (Improvement item ID)
* **Body**: `Idea` (JSON)
* **Description**: Update an existing improvement item.

### 5. **Delete Improvement Item**

* **DELETE** `/api/improvements/{id}`
* **Path Variable**: `id` (Improvement item ID)
* **Description**: Delete an improvement item.

### 6. **Like an Improvement Item**

* **PUT** `/api/improvements/{id}/like`
* **Path Variable**: `id` (Improvement item ID)
* **Description**: Increment the like count for an improvement item.

### 7. **Export Improvement Items to Excel**

* **GET** `/api/improvements/export`
* **Description**: Download all improvement items in an Excel file.

## Data Model

### `Idea`

The main entity, representing an improvement idea, has the following fields:

* `id`: Unique identifier (generated by JPA)
* `startTime`, `completionTime`: Timestamps for when the item started and completed
* `email`: Submitterâ€™s email
* `toilItem`: Short title for the improvement
* `description`: Detailed description of the idea
* `category`: The category of the improvement (e.g., BUG, FEATURE, etc.)
* `impact`: Impact level (High, Medium, Low)
* `status`: Current status (e.g., OPEN, IN_PROGRESS, COMPLETED)
* `priority`: Assigned priority (e.g., LOW, MEDIUM, HIGH)
* `likes`: Number of likes for the idea
* `createdDate`, `updatedDate`: Timestamps for creation and updates
* `jiraCreated`, `jiraId`: Whether a JIRA ticket was created for the improvement

### Enum Types

* `Category`: Categories like BUG, FEATURE, IMPROVEMENT, etc.
* `ImpactLevel`: High, Medium, Low
* `Status`: Open, In-Progress, Completed
* `Priority`: Low, Medium, High, Critical
* `Complexity`: Simple, Moderate, Complex

## Setup & Installation

### Prerequisites

* JDK 17 or above
* Maven (or Gradle)
* IDE (e.g., IntelliJ IDEA, Eclipse)
* A relational database (MySQL, PostgreSQL, etc.)

### Clone the Repository

```bash
git clone https://github.com/yourusername/team-portal-backend.git
```

### Build & Run the Application

1. Navigate to the project directory:

   ```bash
   cd team-portal-backend
   ```

2. Build the application using Maven:

   ```bash
   mvn clean install
   ```

3. Run the Spring Boot application:

   ```bash
   mvn spring-boot:run
   ```

   The application will start on port `8080` by default.

### Configuration

The application can be configured by modifying the `application.properties` or `application.yml` file for:

* Database connection (JPA settings)
* Port number
* Other Spring Boot properties

For database configuration, you can set the following properties in the `application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/team_portal_db
spring.datasource.username=root
spring.datasource.password=yourpassword
spring.jpa.hibernate.ddl-auto=update
```

### Accessing the Application

Once the application is up and running, you can access the endpoints via:

* **Frontend**: Integrate with a frontend (Angular, React, etc.) using the provided API endpoints.
* **Postman/Browser**: You can also test the endpoints using tools like Postman or directly in the browser (for `GET` requests).

## Excel Export

The application supports exporting all improvement items to an Excel file. This can be triggered via the following endpoint:

* **GET** `/api/improvements/export`

The Excel file will be generated at the path `src/main/resources/ideas_data.xlsx` and can be downloaded.
